1. Automata Configuration

The left panel of the Workspace is where you define the core properties of your machine:

  • Type: Choose the class of your machine. DFA (Deterministic Finite Automaton), NFA (Non-Deterministic Finite Automaton), or ENFA (Epsilon-NFA).
  • States: A comma-separated list of your machine's states (e.g., q0, q1, q2).
  • Alphabet: A comma-separated list of valid input symbols (e.g., 0, 1 or a, b).
  • Start State: The single state where execution begins (e.g., q0).
  • Accept States: A comma-separated list of final states that result in a successful match (e.g., q2, q3).

Note: The toolkit features a "Smart Normalizer." If your state names become too long or complex, it will automatically normalize them to a clean q0, q1... namespace while preserving mathematical integrity.

2. The Transition Table

Once your configuration is set, the Transition Table allows you to define the state-to-state routing.

  • For DFAs: Use the dropdown menus. By mathematical definition, a DFA transitions to exactly one state per symbol.
  • For NFAs / e-NFAs: Use the checkboxes. You can select multiple destination states for a single symbol. e-NFAs will feature an additional column for the epsilon (#) transition, representing a state change without consuming an input character.

3. Interactive Graph Visualization

Clicking "Draw Automata" triggers the D3.js physics engine to render your machine.

  • Node Dragging: Click and drag any state (circle) to pin it to a specific location. The physics engine will route the edges around it.
  • Infinite Canvas: If you drag a node to the edge of the white box, the SVG canvas will automatically expand its boundaries to accommodate large graphs.
  • Visual Indicators: Accepting states are denoted by a double-circle. The Start state is denoted by an incoming arrow from nowhere.

4. Time-Travel Tape Simulator

Test if your machine accepts a specific string of characters while visualizing the exact paths taken.

  • Input Word: Enter the string you want to test (e.g., 1011).
  • Reset: Loads the word onto the visual tape. For e-NFAs, it automatically calculates the initial Epsilon Closure.
  • Next / Back: Step through the input character by character. The application stores an execution history array, allowing you to reverse time if you miss a transition.

Simulator Color Legend:

  • Active Path: The currently active state(s) and transition edges.
  • Pending: The next character in the input sequence.
  • Current: The character most recently evaluated.
  • Processed: Characters that have already been evaluated.

5. Conversions and Mathematical Operations

The toolkit can automatically convert machines using advanced computer science algorithms:

  • Convert to NFA: (For e-NFAs) Removes epsilon transitions and computes extended delta transition paths.
  • Convert to DFA: (For NFAs) Uses the formal Subset Construction algorithm to create a deterministic equivalent.
  • Minimize DFA: Uses Partition Refinement (k-equivalence classes) to merge identical states and reduce the DFA to its absolute minimal form.
  • Convert to Regex: Uses the GNFA (Generalized NFA) State Elimination method to derive a Regular Expression.
  • Complement DFA: Inverts the DFA. It first strictly "completes" the DFA (injecting a deadState to ensure a Total Transition Function), then swaps accepting and non-accepting states.

6. Regular Expression Engine

Work directly with Regular Expressions using the inputs at the top of the Workspace.

  • Convert Regex to eNFA: Compiles a regex string into an e-NFA using Dijkstra's Shunting-yard algorithm and Thompson's Construction.
  • Complement Regex: A powerful macro that takes a regex, converts it to an e-NFA, makes it deterministic, minimizes it, complements it, and converts it back into a new Regex representing the inverted language!

Regex Operator Precedence & Rules:

  1. Parentheses (): Highest precedence for grouping.
  2. Kleene Star *: Zero or more occurrences.
  3. Concatenation: Implicit (e.g., ab means a followed by b).
  4. Union |: Lowest precedence (Logical OR).

Note: The engine mathematically simplifies outputs, handling empty sets and epsilon identities. Crucial: Do not use the . character for concatenation; it is handled implicitly.

7. Automaton Versions (History Cache)

As you convert machines (e.g., e-NFA ➔ NFA ➔ DFA ➔ Minimized DFA), the application caches the intermediate steps in memory. Use the Automaton Versions buttons to instantly restore and view a previous historical state without having to re-type it.

8. Language Equivalence

The Test Equivalence view mathematically proves if two different machines (or Regular Expressions) describe the exact same language.

  • Methodology: The engine normalizes both inputs into Minimal DFAs and performs a BFS across their Cartesian Product.
  • Inputs: For Representation A and B, you can either upload a .txt file or directly type a Regex.
  • Isomorphism Mapping: If the languages match, it displays an Isomorphism map, showing exactly how the minimized states of Machine A mathematically map 1:1 to the minimized states of Machine B.
  • Shortest Counter-example: If they differ, the BFS guarantees finding the shortest possible string that one machine accepts while the other rejects.

9. File Import, Export, and Syntax

Save your progress natively to your local machine.

  • Export SVG / PNG: Downloads a clean image of your graph.
  • Export TXT: Downloads a text file containing your automaton's raw mathematical definition.
  • Import TXT: Upload a text file. The parser has a smart fallback: if the file does not match the strict Automaton syntax, it will attempt to read the file's contents as a raw Regular Expression string.

TXT File Format Syntax

To manually write or edit an imported .txt file, follow this exact structure:

TYPE: DFA
STATES: q0, q1, q2
ALPHABET: 0, 1
START: q0
ACCEPT: q2
TRANSITIONS:
q0 0 q0
q0 1 q1
q1 0 q2
q1 1 q1
q2 0 q2
q2 1 q2

Note: Use the # character for epsilon transitions. Ensure the transition lines follow the [fromState] [symbol] [toState] format strictly.

10. Troubleshooting and FAQ

Q: Why is my DFA conversion taking a long time?
A: Subset construction can lead to an exponential "state explosion" (up to 2n states). Ensure your NFA is as simple as possible.

Q: The graph nodes are overlapping. How do I fix this?
A: Click and drag the nodes to rearrange them. The physics engine will maintain your custom layout until the next redraw.

Q: I imported a file but the transition table is empty.
A: Check that your TXT file matches the syntax in Section 9 exactly, including the "TRANSITIONS:" header.
